### 🛠 技术讨论：AI 编程工具流与模型选择

**1. 开发工作流新共识：Gemini 3 负责“白板”，Cursor 负责“落地”**

* **组合打法：** 多位群友验证了一种高效模式——先利用 **Google AI Studio (Gemini 3)** 的“白板”或“口喷”模式快速生成 Demo、验证核心思路（速度快、Bug少）；然后将生成的代码下载，转入 **Cursor** 进行细致开发和功能完善。

* **关于 Cursor 的地位：** 尽管新工具层出不穷（如 Windsurf、Bolt 等），小排老师及多位资深群友建议**新手先守住 Cursor**。Cursor 的上限很高，跑通全流程是首要任务，不要盲目切换工具导致精力分散。

* **模型性价比分析：**

  * **Kimi K2：** 烧钱速度较快，有群友反馈“一个周末烧掉100多”，建议关注包月套餐。

  * **OpenAI Codex：** 被认为是目前“耐用”的选择，200美元/月的额度很难用完，且代码风格偏向源码，适合配合 Cursor 使用。

  * **Claude Code (cc)：** 代码可读性好，但中转 API 成本较高，建议在需求明确后再调用。

### 🚧 实战避坑：基础设施配置指南

**1. 谷歌账号（Gmail）注册与验证难题**

* **痛点：** 国内手机号验证失败、提示“无法验证”或“次数过多”。

* **群友验证的解决方案：**

  * **环境设置：** 将浏览器语言设置为英语，配合干净的 IP，成功率较高。

  * **接码技巧：** 使用接码平台（如 5sim）时，若短时间失败多次请暂停，避免账号被风控。有群友推荐 **Giffgaff** 实体卡注册 Droid 等服务。

  * **替代方案：** 实在注册不下来的，建议直接在交易平台（如闲鱼）购买成品号，或使用无需手机验证的旧版入口（运气成分）。

**2. Supabase 数据库配置排雷**

* **连接失败：** 许多群友遇到 Cursor 连接 Supabase 失败的问题。**核心原因**通常是免费账户的 Project 数量达到上限（通常为 2 个），删除多余项目后即可恢复连接。

* **Key 的混淆：** 区分 `API Key` 和 `Anon Key`。在 `.env` 文件配置时，务必检查 `DATABASE_URL` 和 `NEXT_PUBLIC_SUPABASE_ANON_KEY` 是否填写正确，不要混用。

* **用户权限（Role）管理：**

  * 不要试图在 Supabase 后台手动穷举权限。

  * **正确做法：** 在需求文档中明确定义逻辑（例如：“未登录用户只能用1次，登录用户免费2次，付费用户无限”），然后指示 Cursor 通过代码逻辑（如字段 `is_admin` 或 RLS 策略）来实现。

**3. Vercel 部署问题**

* **自动部署失效：** 通常是因为环境变量（Environment Variables）未在 Vercel 后台正确配置。

* **解决方法：** 确保 Github 推送正常，并在 Vercel 项目设置中补全所有 `.env` 里的关键变量。

### 📝 总结：给没看群消息的你

1. **别纠结工具：** 新手锁死 **Cursor**，想搞快 Demo 试玩就去用 **Google AI Studio**。

2. **搞定基建：** 注册谷歌号遇到手机验证不过，优先换浏览器语言为**英语**或直接买号；Supabase 连不上先检查是不是**项目建多了**。

3. **需求先行：** 遇到权限控制等复杂逻辑，先写清楚**需求文档**（PRD）喂给 AI，而不是自己去数据库后台瞎改配置。

4. **关注小排直播：** 学习产品分析方法论，利用 RAG 技术解决垂直领域知识问答精度问题。
