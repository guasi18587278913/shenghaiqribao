### 🛠 技术实战：支付逻辑与工具避坑

**1. Creem 订阅升级的“差价抵扣”如何实现？**
针对“用户在半途升级套餐，如何计算并展示抵扣金额”的问题，群内展开了深度讨论，形成了两条技术路径：

* **API 组合拳方案（推荐）：** 核心逻辑由开发者系统控制。

  * 第一步：在自家系统算出应抵扣金额（Credit）；

  * 第二步：调用 Creem API 创建一个固定金额的 Discount（折扣码），绑定到目标产品；

  * 第三步：创建 Checkout Session 时预填该折扣码。

  * *优势：* 灵活控制抵扣逻辑，前端可明确展示“原价-抵扣=实付”。

* **官方门户方案：** 使用 Creem 的 Customer Portal API/SDK。

  * 通过 `createPortal` 接口生成链接，让用户在官方托管页面完成升级，系统会自动处理折算预览。

**2. Claude Code 长文本“卡死”与“截断”的解决方法**
多位群友反馈使用 `claude code` 时遇到严重 bug：

* **现象：** 直接粘贴长文本（如 5000 字），系统可能会静默截断（只读入前 2500 字而不报错），或者直接卡死。

* **排查技巧：** 可以要求 AI 输出“接收到的文本最后 10 个字符”来验证完整性。

* **解决方案：** 避免直接粘贴。将长内容保存为 `.md` 文档，使用指令让 Claude 直接读取该文档文件。

**3. 开发工具链琐事**

* **Supabase 邮件风控：** 有群友反馈 Supabase 因“退信率太高”发警告。建议方案是切换到 **Resend** 服务，并优化邮箱模板。

* **OpenCode 兼容性：** 字节系的 OpenCode 更新后，部分 Windows 和 Mac 用户出现无法使用的情况，但也有 Mac 用户反馈正常。

* **API 资源：** 针对 OpenRouter 国内连接不稳定的问题，以及寻找便宜的视频生成 API（如 Sora 2 类），群友推荐关注 `evolink.ai`、`kie.ai`、`apimart.ai` 等中转/聚合平台。

### 🌍 跨境业务：合规与账户体系

**1. 跨境支付账户新动态**

* **PayPal 个人卖家账户上线：** 官方推出了个人卖家账户，支持分钟级注册，利好个人出海收单。

* **Stripe 多站管理：** 确认了在海外注册一家公司主体下，可以开通 Stripe 并挂载多个不同类型的网站进行收款。

**2. 税务合规预警**

* 转发了关于税务部门对“近三年境外所得开展自查”的提醒文章，建议涉及境外收入的群友关注合规风险。

### 📝 总结

1. **开发警示：** 用 Claude Code 处理长代码/文本务必通过文件读取，谨防粘贴时的静默截断。

2. **支付集成：** 处理订阅升级差价，优先考虑“计算差额生成一次性折扣码”的方案，用户体验更可控。

3. **基础设施：** 邮件服务若遇风控可切 Resend；OpenCode 近期更新有兼容性波动。

4. **合规风向：** 个人出海收单渠道变宽（PayPal），但境外收入的税务合规审查正在收紧。
