### 01 工具之争：Cursor 还是 Google Antigravity？

**Cursor 的去留问题**

* **争议点：** 部分高频用户（如 arlen）认为 Cursor 性价比变低，Token 消耗极快（“每个月都超标”），打算取消订阅改用 API 组合（Claude Code + 智谱）；但也有用户（如简单）坚持续费，认为对于体系化、快速开发，Cursor 的出字速度和丝滑程度依然是首选，“用来调 Bug 还是好用”。

* **设置建议：** 有用户建议关闭 `on-demand` 模式，只保留 `auto` 模式以节省成本；但反方观点认为 `auto` 模型相对较“傻”，只适合单纯执行，不适合复杂任务。

**Google Antigravity (及反代方案) 的优劣**

* **成本优势：** 相比 Cursor 的官方订阅，通过组建 Google 家庭组（Family Plan）不仅成本极低（甚至低至百元/年），且 Ultra 套餐似乎不限额（Pro 和免费版有限制）。

* **当前短板：**

  * **逻辑缺陷：** 多位用户反馈，在 Antigravity 中使用 Opus 4.5 的效果不如直接用 CC（Claude Code/反代服务），代码 Review 逻辑问题多，需要二次修复。

  * **迭代缓慢：** 相比 Cursor 的高频更新，Google 此项目更新慢，Bug 较多，被认为“精力主要在收集数据而非优化体验”。

* **结论：** 如果追求极致稳定和丝滑，Cursor 依然是王者；如果追求极致性价比且愿意折腾账号，Google 家庭组方案更优。

### 02 实战技巧：提示词与环境配置

**让 Gemini 乖乖说中文**

* 在 Antigravity 中，Gemini 经常不用中文思考和回答。群友 @Shine-厦门 分享了一个强制中文的 Prompt 配置，亲测有效：

```yaml

## trigger: always_on
alwaysApply: true
priority: highest

## (在此处添加要求中文思考的指令)
```

**Windows 环境下的登录与连接问题**

* **故障现象：** Windows 登录某些 AI 服务（如 Claude/Google 对应服务）卡住跳不过去。

* **解决方案：**

  1. 确保网络环境开启全局代理 + TUN 模式；

  2. **关键点：** 将系统的“时间和语言/区域”设置为英语/美国；

  3. 如果仍不行，使用 Proxifier 强制接管流量；

  4. 最后排查是否为账号本身的风控问题。

### 03 资源与“黑科技”：账号获取与管理

**“家庭组”与多账号策略**

* **玩法揭秘：** 为了应对 Token 限额，部分硬核玩家采用了“号海战术”。一人注册 15+ 个 Google 账号，通过 3 个主号建立家庭组，拉入大量小号（Student Accounts）共享额度。

* **注意事项：**

  * 主号最好是美区，且邀请时小号需挂美区 IP 接受邀请。

  * 手机号注册限制：通常一个手机号只能注册约 3 个 Google 账号，可以通过接码平台或购买小号邮箱解决。

**闲鱼搜索暗号**

* 由于平台风控，“ChatGPT”已成为闲鱼敏感词无法搜索。

* **替代关键词：** 搜索 **“codex team”** 或类似技术团队词汇，可以找到相关账号或拼车服务。

### 📝 总结与行动建议

1. **Cursor 用户自查：** 检查你的用量，如果 Token 经常溢出且预算敏感，可尝试关闭 `on-demand` 或考察 API 组合方案（Claude Code + 国产模型）。

2. **Google 方案尝鲜：** 如果你想低成本使用高级模型，可以尝试组建 Google 家庭组，但要做好应对 Bug 和逻辑问题的心理准备。

3. **Prompt 优化：** 在使用 Gemini 等模型时，尝试加入 `priority: highest` 类配置强制规范输出语言。

4. **账号购买技巧：** 在二手平台寻找 AI 服务时，请使用技术黑话（如 codex team）绕过关键词屏蔽。
