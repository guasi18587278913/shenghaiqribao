### 💡 提效新招：Claude Code 的“反问”艺术

刘小排分享了一个 Claude Code（CC）中鲜为人知但极其实用的功能——**`askuserquestion`**。

* **核心价值：** 在开发者只有模糊想法、需求不清晰时，CC 不会盲目写代码，而是利用该功能主动向用户提问，引导开发者一步步厘清思路。有群友反馈，这解决了“有想法但描述不清”的痛点，通过几轮对话即可明确开发路径。

* **替代方案：** 针对没有 CC 权限的用户，群友 **Moore 助教** 提出 GPT 系列也能通过 Prompt 达到类似效果。建议使用 `gpt5.2 -think`（假设的指令或特定配置），并在 Prompt 中要求模型遵循“渐进式披露”原则，逐步展开询问，而非一次性输出代码。

### 🚨 工具动态：Antigravity 模型大面积“罢工”与修复

今日下午，多个群组集中反馈 **Antigravity**（反重力模型，常见于 Cursor 等 AI 编辑器）无法加载、报错 529 或一直处于未登录状态。

* **故障现象：** 代码写到一半中断、模型加载失败、提示未登录。

* **排查结论：** 官方似乎出现了短暂的不稳定（有群友分享了小红书上的官方动态），同时该模型对网络环境要求极高。

* **解决方案（实测有效）：**

  1. **节点选择：** 普通节点可能失效，建议切换至 **美国住宅 IP**（家宽）节点。

  2. **网络模式：** 仅切换节点可能无效，必须检查代理软件设置。群友 **Newbie\_成都** 建议开启 **TUN 模式** 并给予管理员权限，确保流量被正确接管。多位群友在开启 TUN 模式后恢复正常。

### ⚠️ 避坑指南：一次惨痛的“删库”经历

群友 **小李哥** 分享了一次生产环境事故：在使用数据库工具时，误执行了 `db:push` 命令。

* **事故还原：** 本地环境直接将架构/数据 push 到了生产环境数据库，导致原有数据被覆盖/清空。

* **后果：** 几十篇有流量的原创博文数据丢失。

* **恢复尝试：** 使用的云数据库服务商 **Neon** 的免费版/当前层级仅支持恢复 **6 小时前** 的数据，无法完全挽回损失。

* **警示：**

  * 严格区分 `db:push`（通常用于同步 schema，具破坏性）和 `db:migrate`。

  * 生产环境操作前务必做好全量备份。

  * Supabase 等类似服务的恢复功能通常需要付费会员支持。

### 🛠️ 资源与杂谈：账号购买与前端新工具

1. **Google/Cursor 账号避坑：**

   * 关于购买 Google Pro / Cursor Pro 账号，群友建议优先选择 **学生认证** 版本而非家庭组共享。

   * **争议点：** 有观点认为家庭组共享额度会导致每个人可用量减少（存在风险），而学生包相对独立且稳定。

2. **Coze 编程限制：** 有群友尝试在 Coze（扣子）中使用 Python 环境，发现无法像本地电脑一样随意调用环境。**智昊** 指出 Coze 是沙箱环境，出于安全考虑做了大量隔离，无法通过常规手段绕过。

3. **前端设计画布：** **李澹归** 推荐了 `pencil.dev`，这是一款近期在 Twitter 上爆火的前端设计工具，适合从设计直接转代码。

### 📝 总结：今日关键收获

1. **AI 交互技巧：** 善用 Claude Code 的“反问”模式（`askuserquestion`），能大幅提升需求清晰度；GPT 可通过 Prompt 模仿此效果。

2. **网络调试经验：** AI 工具（如 Antigravity）若出现诡异连接问题，优先检查是否开启了代理软件的 **TUN 模式**。

3. **运维安全：** 操作生产数据库时，严禁随意使用 `push` 类覆盖命令，且务必确认云厂商的数据恢复策略（Time Point Recovery）是否覆盖当前套餐。
