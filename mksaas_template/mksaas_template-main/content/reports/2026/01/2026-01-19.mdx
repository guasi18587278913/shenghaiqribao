### 🚀 技术实战：Claude Code 配置与移动端编程

**1. Claude Code/Codex 效率提升与指令变更**

* **Ultrathink 指令失效：** 多位群友确认 `ultrathink` 触发最大推理能力的指令已被官方弃用。现在的解决方案是手动设置环境变量。

  * **操作方法：** 使用命令 `export MAX_THINKING_TOKENS=63999`，然后运行 `claude --dangerously-skip-permissions`。

* **性能感知：** 大家普遍反馈近期 Codex 响应速度显著变快，Token 消耗似乎有所优化（得益于 LSP 将代码 AST 化，减少了无关 Token 的读取）。

* **移动端编程新姿势：** 刘小排分享了在手机上通过 Termux 等工具运行 Claude Code 和 Codex 的方案，实现了“走路也能编程”的设想。不过有群友反馈使用 Happy Codex 时遇到了 `-32001 Request timed out` 的报错，可能需要网络环境调试。

**2. Next.js 开发新资源**

* **Skills 规范库：** 助教分享了一个专门针对 Next.js 的 Claude Skills 规范库，能有效提升代码生成的健壮性，建议写 Next.js 的同学接入使用。

**3. Google/Gemini 账号风控讨论**

* **Antigravity 报错：** 有群友在使用 antigravity tool 管理 Gemini Pro 账号时，遇到多思考一会儿就报错的问题。

* **排查思路：**

  * **节点问题：** 建议挂链式代理，保持 IP 稳定；

  * **家庭组权限：** 确认主账号是否开启了 Google One 共享；

  * **工具风控：** 社区有反馈 antigravity tool 可能被谷歌风控，建议暂时停用或安装 `Antigravity Cockpit` 插件尝试解决。

### ⚠️ 合规与支付：AI 小程序备案与 PayPal 接入

**1. AI 小程序过审“血泪史”**

* **核心教训：** 做 AI 类小程序，**一定要提早进行互联网信息服务备案**。

* **正确流程：**

  1. 先提交互联网信息服务备案（最快约 5 天）；

  2. 备案通过后，在小程序后台设置中增加“服务类目”，并上传备案通过截图；

  3. 最后才提交小程序代码审核。

* **避坑提醒：** 如果没有完成上述步骤直接送审，极大概率会被无限打回。目前国内对 AI 类小程序审核非常严格，虽有“付费快速过审”的说法，但走正规流程更稳妥。

**2. PayPal 个人账户接入差异**

* **接入实操：** 有群友在接入 PayPal 个人账户时发现，与 Stripe 不同，PayPal 似乎不需要预先创建用户订单，可以直接在代码中编写逻辑。刘小排指出，如果学通了课程里的 Stripe (Creem) 接入，PayPal 本质上是一样的，无需专门教程。
