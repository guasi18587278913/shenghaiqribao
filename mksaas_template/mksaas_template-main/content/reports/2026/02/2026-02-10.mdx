### 💸 成本控制：如何避免“反重力”API 瞬间烧干？

OpenClaw 虽强，但 Token 消耗速度惊人。群友们总结出了一套“高性价比”的模型组合策略，拒绝只用昂贵的 Claude Opus 硬抗：

* **模型分工策略：**

  * **日常沟通/简单探索：** 强烈推荐使用 **Gemini Flash** 或 **Grok**。Gemini Flash 极其便宜且皮实，Grok 通过中转站调用成本低至 0.7元/百万Token。

  * **核心任务执行：** 遇到写代码、复杂逻辑推理时，再切换回 **Codex** 或 **Opus**。

  * **原则：** “不要把 Token 浪费在探索上，要把好钢用在提升效率的刀刃上。”

* **关于 Opus 4.6 灰度：** 有群友发现“反重力”（某中转服务）的 Ultra 版本已支持 Opus 4.6 (1M context) 的灰度测试，但 Pro 版本尚未全量开放。

* **注意事项：** 中转 API 虽便宜，但在高强度任务（全量 Opus）下依然昂贵，且存在缓存计算差异，长期重度使用官方订阅或许更稳。

### 🚧 平台选择：飞书 API 踩坑与 Telegram 注册“野路子”

在将 OpenClaw 接入聊天软件时，群友们通过血泪教训得出了最优解：

**1. 飞书（Feishu）的致命短板**

* **额度陷阱：** 多位实战群友反馈，飞书**个人基础版 API 有单月 1万次 调度的硬性限制**。

* **后果：** 高频使用下，额度通常在 **2天内** 就会耗尽，机器人直接“失联”或不回话。

* **结论：** 除非拥有企业版高额度，否则不建议个人用户将主力机器人部署在飞书。

**2. Telegram（TG）的曲线救国注册法**
鉴于飞书的限制，Telegram 成为首选，但注册难度较高（收不到验证码）。群内大神分享了一套**iPhone 必成注册流程**：

* **环境准备：** 使用美国 AppStore 下载 TG App，节点切换至日本或美国等海外纯净 IP。

* **关键步骤：**

  1. 使用国内手机号注册。

  2. App 会要求购买 **TG 一周会员（约 $1.99）** 才能发送短信验证码。

  3) **重点：** 购买后，前两次短信验证通常依然会失败，此时系统会转为**电话语音播报验证码**（会报两次）。

  4) 接听电话输入即可成功。
     *注：支付可购买美区礼品卡解决。*

### 🛠️ 技术排查：部署报错与环境配置

针对 OpenClaw 部署中的常见报错，群内输出了具体解决方案：

* **报错：`memory_search` 缺 API Key**

  * **原因：** Codex OAuth 仅用于对话，不包含 embedding 权限。

  * **解法：** 需要单独配置 embedding key。可以选择让其下载 Local 本地模型，或者接入 Gemini / Voyage 的 embedding 模型。

* **报错：`issue with selected model (claude-opus-[1m])`**

  * **原因：** 反代服务（如反重力）暂不支持或仅灰度支持 1M 上下文模型，或者选择了不存在的模型 ID。

  * **解法：** 切换回标准的 opus 模型即可恢复。

* **自动化被墙问题：**

  * OpenClaw 在进行网页抓取或自动化操作时，极易因 IP 问题被目标网站（如 X 平台）拦截。

  * **VPS 困境：** 海外 VPS 访问国内站被墙，国内 VPS 访问海外站需梯子。

  * **替代方案：** 相比云服务器，有群友建议使用 **Mac Mini** 配合住宅 IP 放在本地跑，环境更拟人，封号风险相对较低。

**📝 总结：**

1. **省钱法则：** 简单对话用 Gemini Flash/Grok，复杂任务用 Codex/Opus。

2. **避坑指南：** 个人玩 OpenClaw **别用飞书**，额度两天就没；推荐转战 Telegram。

3) **技术配置：** 记得单独配置 Embedding Key 以启用记忆搜索功能；遇到模型报错先检查是否使用了未开放的灰度模型。

4) **环境建议：** 自动化任务对 IP 纯净度要求极高，云服务器容易被风控，本地 Mac Mini 是个不错的替代思路。
