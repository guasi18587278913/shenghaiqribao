### 01. 重磅发布：GPT-5.4 上线与实测反馈

**GPT-5.4 初步评价**
GPT-5.4 及 Pro 版本今日发布，引发群内热议。

* **能力提升：** 多位成员反馈 5.4 终于“能说人话了”，相比之前的版本，语言表达更加清晰自然，不再是生硬的汉字堆砌。
* **与 Claude 对比：**
* 在浅层推理、快速任务上，Claude Opus/Sonnet 依然有低延迟优势；
* 在深度推理、复杂项目规划及 Agent 任务中，GPT-5.4 表现出更强的规划能力（“Claude 还在想，GPT 已经干完”）。
* **兼容性：** OpenClaw 目前可通过 OAuth 授权或反向 API 支持 GPT-5.4；Codex 用户也可以直接调用。

### 03. 技术实战：工具配置与避坑指南

**Claude Code (CC) 使用技巧**

* **Token 爆炸预警：** 有群友使用 Claude Code 时 Token 消耗极快（甚至达到 7.3M 上下文）。
* **原因：** 长期未清理上下文，导致历史记忆累积。
* **解决：** 养成使用 /clear 命令清理上下文的习惯，或编写脚本限制。
* **调用 Codex：** 在 CC 中调用 Codex 或 Gemini，推荐使用 GuDaStudio/commands 插件。
**网络配置与规则优化 (Clash/Surge)**

* **TUN 模式的重要性：** 多位群友反馈，不开 TUN 模式会导致部分软件（如 antigravity、模型列表加载）无法正常工作。建议常驻 TUN 模式。
* **规则集推荐：**
* **Loyalsoldier：** 被公认为最简单、稳健、开箱即用的方案，适合不想折腾的用户。
* **MetaCubeX/Dat文件：** 适合喜欢自定义的高级用户，规则粒度更细，但配置相对繁琐。
* **共识：** 对于绝大多数用户，引用成熟的在线规则集（如 Loyalsoldier）比自己手搓规则更高效。
