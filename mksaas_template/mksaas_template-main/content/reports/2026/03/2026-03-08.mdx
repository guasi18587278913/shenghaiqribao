### 🚀 技术风向：OpenClaw 新功能与模型实战红黑榜

**1. OpenClaw 3.7 更新“Binding Model”：单窗口变身“多部门办公室”**
Moore 助教详细解读了 OpenClaw 的新功能，核心理念是将 Discord 或 Telegram 的特定对话变成“专属工位”。

* **多任务玩法：** 可以在不同的对话窗口绑定不同的模型和工作目录。
* 例如：窗口 A 绑定 Codex 负责写 Python 代码；窗口 B 绑定 Claude Code 负责写文档；窗口 C 绑定 Gemini 处理一次性脚本。
* **单窗口多任务（老板模式）：** 通过 /acp 系列指令，可以在同一个对话中临时切换模型和上下文。
* **形象比喻：** 用户是老板，同一个聊天框是办公室。你可以对 A 说“写页面”，对 B 说“写接口”，对 C 说“做测试”。他们互不干扰，但都在你的掌控中。
* **指令技巧：** /acp model 切换模型，/acp cwd 切换工作目录，任务结束后 /acp reset-options 复原。
**2. 模型实战红黑榜（PinchBench 更新）**
围绕最新的模型排行，群友们分享了真实的实测体验：

* **红榜（推荐）：** **Opus 4.6** 被多位群友公认为目前最强、最顺手的模型，甚至优于 GPT-5.4。
* **黑榜（避雷）：** **MiniMax** 被指出指令遵循能力较差，且存在严重的“幻觉”问题。
* **开发痛点：** 有群友反馈，在做微信小程序开发时，无论是 Claude Code 还是 GPT，对前端界面的还原度都很差（图片、布局跑偏），目前的解决方案是先用 Nano Banana 生成前端图，再让大模型参照开发。
**3. 解决 OpenClaw 生成飞书文档的“无权编辑”问题**
多位群友遇到小龙虾（OpenClaw）生成的飞书文档或多维表格，人类用户只有阅读权、没有管理权的问题。

* **解决方案：** 并不是系统 bug，而是需要通过特定的提示词或更改 openclaw.json 配置文件来确权。群内已分享相关脱敏版的权限管理方案文档。

### 👥 社群运营：深海圈续费机制与课程权益

**1. 核心争议：过期后能否观看历史课程？**
关于“深海圈”AI 编程课程的有效期问题，官方（亦仁、欢欢）明确了以下规则，引发了部分学员的担忧：

* **机制：** 深海圈按“社区”运营而非单纯卖课。会员期内可免费看所有新旧课程；**一旦会员过期，将无法登录社区，即无法观看任何课程（包括已买的第一期内容）。**
* **用户顾虑：** 部分学员认为自己购买的是课程，过期不能复习不太合理；且实战中往往需要“边做边查”，担心没时间学完。
* **官方回应：**
* 此设定是为了倒逼大家“抓紧学、赶紧用”，避免囤课。
* 只要续费（保持在圈内），所有未来更新的几期课程都能免费看。
* 后续会推出针对老学员的“骨折级”续费优惠方案。
**2. 其它运营动态**

* **分销：** AI 编程深海圈将上线分销机制（CPS），有需求可联系官方获取资料。
* **虚拟分身：** “虚拟小排老师”正在开发中，未来可能上线辅助解答问题。
