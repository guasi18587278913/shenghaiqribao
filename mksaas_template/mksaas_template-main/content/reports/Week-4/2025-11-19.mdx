### 🔥 重磅炸弹：Gemini 3 Pro 与 Antigravity 初体验



Google 发布的 Gemini 3 Pro 模型及官方 IDE 工具 Antigravity 引发了全员测试热潮。



* **能力评价：** 普遍反馈模型能力极强，甚至用“变态”、“吓人”来形容。特别是对于前端代码的多层嵌套和数据传递，Gemini 3 Pro 能一次搞定 Codex 难以处理的需求。有群友感叹“前端程序员真的不存在了”，未来大家将转型为“拿着 AI 鞭打代码”的 Builder。

* **V0 更新：** Vercel 的 V0 工具也迅速跟进，已支持切换至 Gemini 3 模型，体验被评价为“更上一层楼”。

* **当前槽点：**

  * **限流严重：** 许多用户反馈模型并不耐用，跑一会儿就会提示限额或 Overload（过载）。

  * **响应慢：** 虽然代码质量高，但生成时间较长，且 IDE 在运行过程中容易出现 "Agent terminated" 报错。



### 🛠️ 避坑指南：Antigravity 登录失败与卡顿解决方案



超过半数参与讨论的群友都卡在了 Antigravity 的登录环节（无限转圈、Next 按钮不可点、地区不支持）。以下是群友摸索出的**成功解决方案汇总**：



1. **换账号（首选方案）：**

   * 大量案例证明，老账号容易被风控或卡死。

   * **解决方案：** 注册或切换一个新的 Google 账号（非 Pro 会员账号也能登录，甚至成功率更高）。

2. **网络环境配置：**

   * 必须开启代理的 **TUN 模式** 或 **全局模式**。

   * 建议使用纯净的美国 IP（住宅 IP 最佳），部分节点可能导致地区不支持错误。

3) **玄学操作：**

   * 若卡在登录页，尝试流程：退出账号 -> 杀掉程序进程 -> 重新打开程序 -> 登录 -> 退出 -> 再登录。

   * 检查 Google 账号管理中的“年龄认证”，确保账号信息完整。

4) **关于手机号验证：**

   * IDE 内部验证不支持 +86 手机号，建议使用接码平台或非 +86 号码。



### ⚔️ 诸神之战：Gemini 3 vs GPT-5 vs Cursor



随着新模型的发布，群内对“目前最好用的编程组合”展开了讨论：



* **代码生成首选：** 刘小排等核心成员认为，在纯写代码层面，**Gemini 3 Pro 目前是最强的**，建议在 Cursor 中调用 Gemini 3 Pro 模型进行开发。

* **对话与逻辑首选：** 在产品逻辑沟通、需求分析和日常对话上，**GPT-5 Pro** 依然被认为是"王者"，精准度极高，能从文档深处挖掘信息。

* **工具搭配建议：**

  * **小白/产品经理：** 推荐直接使用 Google Antigravity（需克服登录门槛），因为它加入了 Planning 确认逻辑，适合查漏补缺。

  * **开发者：** 维持 Cursor 为主力编辑器，模型选择 Gemini 3 Pro 进行代码编写，遇到复杂逻辑问题切换回 GPT-5 Pro 咨询。
