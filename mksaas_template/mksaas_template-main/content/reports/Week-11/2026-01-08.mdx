### 1. 极限薅羊毛：GPT Plus "老兵认证"与 Gemini 家庭组

今日群内讨论热度最高的话题集中在两个低成本获取顶级模型服务的方案上：

* **ChatGPT Plus "0元购"（高风险）**：

  * **操作原理**：利用美国退伍军人（Veterans）的身份认证渠道，通过 Discord 机器人验证后，可获得 1 年免费 Plus 会员资格。

  * **现状与门槛**：目前已有群友实测成功，但入口似乎开始收紧（显示系统不可用）。海鲜市场（闲鱼）出现了约 30 元的“认证+绑卡”一条龙服务。

  * **风险提示**：多位群友提醒，此方法存在封号风险，建议使用新号操作，且对 IP 环境要求较高。部分群友反馈使用半年未封，属于“富贵险中求”。

* **Gemini 家庭组“无限杯”方案（推荐）**：

  * **思路**：利用学生邮箱开通 Google Workspace 或相关优惠，组建家庭组。一个 Pro 账号可带动 5 个子账号，通过反代技术（Reverse Proxy）将这些账号的配额整合。

  * **优势**：相比 GPT 风控更低，且通过特定工具（如 Antigravity）可以调用 Opus 等模型，实现“量大管饱”。

### 2. 硬核配置：低成本 VPS 搭建 AI 代理服务

围绕如何更稳定、丝滑地使用 AI 编程工具（如 Cursor、Antigravity），群内展开了技术层面的深度交流：

* **本地 vs 云端部署**：

  * 虽然部分群友选择在本地（如 Mac 上的 Brew 服务）挂载代理工具，但**共识倾向于云端部署**。

  * **推荐方案**：购买新加坡节点的腾讯云 VPS（提及 99元/年套餐），部署 `CLIProxyAPI` 等反代工具。

  * **理由**：云端部署不仅实现了 IP 的独立和稳定（避免机场 IP 脏导致被封），还能实现多设备（如 Cursor、网页端）共用同一个反代接口，彻底“闭环”。

* **网络排查技巧**：

  * 针对 Antigravity 加载模型卡顿、Codex 登录失败等问题，核心解决方案是检查网络环境。多位群友验证，开启代理软件的 **TUN 模式** 是解决此类连接问题的关键。

### 3. 实战开发：生产环境下的 API 选型策略

针对群友提出的“线上产品使用书虫（第三方代理）偶尔超时，需寻找更稳定 API”的需求，大家分享了实战经验：

* **稳定性分级**：

  * **第一梯队（最稳）**：官方直连 API。虽然价格无优势，但稳定性最高。

  * **第二梯队（推荐）**：OpenRouter、PipeLLM。被多位开发者推荐用于生产环境，价格与官方持平或略低，但整合性好。

  * **避坑指南**：不要在生产环境使用免费或极其廉价的“逆向/中转”接口（如部分号称免费的 route），这些通常仅适合测试，高并发下极易崩溃。

* **架构建议**：

  * 不要依赖单一渠道。**最佳实践**是在代码层面做降级策略（Fallback）：首选高性价比代理，一旦超时或报错，自动切换到备用的官方 API 或 OpenRouter，确保用户体验不中断。

### 📝 总结

今天的讨论非常务实，从\*\*“怎么白嫖/低成本获取算力”**进阶到了**“怎么稳定地把算力用出去”\*\*。

1. **想省钱**：可以尝试 Gemini 家庭组反代方案，比 GPT 羊毛更稳。

2. **想稳定**：做产品务必做 API 降级策略，不要把业务命脉全押在一个第三方代理上。

3. **搞环境**：VPS + Docker + 反代工具是目前解决 AI 工具网络问题的终极方案。
