### 🛠 技术实战：AI 编程工作流与 OpenCode

**1. 多模型“组合拳”工作流**
针对单一模型能力的局限，资深群友总结了一套高性价比的混合开发流程：

* **GLM (智谱)：** 负责补位和基础任务；

* **Gemini 3：** 专门负责前端代码生成；

* **Claude (Sonnet/Opus)：** 负责后端逻辑编写及 Code Review（代码审查）；

* **GPT-4.7/Codex：** 也是主力选项，有反馈称 GPT-4.7 的指令遵从能力已接近 Sonnet 4.5，且 Codex 模型耐用、成本低。

**2. OpenCode 配置与 LSP 能力**
关于开源编程工具 **OpenCode**，群内确认了其具备 **LSP (Language Server Protocol)** 能力，能够像 Claude Code 一样读取和理解代码库，不再单纯依赖外部 MCP 工具。

* **显示问题：** 若在 IDE 终端显示不全，建议拉宽窗口。

* **封号风险：** 讨论指出 OpenCode 若适配官方 Auth（认证），使用 Claude 渠道（cc）存在较高的封号风险，而其他两家模型相对安全。

**3. 知识库对接**
关于“如何将飞书 Wiki 作为知识库投喂给 Codex”，群友建议利用字节系的 **火山方舟（Volcano Ark）**，其自带知识库 API，能较好地解决企业文档与编程工具的互通问题。

### ⚠️ 避坑指南：账号订阅与网络故障排查

**1. Google/Antigravity 订阅失败**
多位群友遇到 Google 账号无法订阅 Antigravity 或 Gemini 会员的问题（页面打不开或报错）。

* **原因分析：** 可能是账号年龄未验证（未满18岁）、IP 节点被风控、或该账号曾领取过免费权益。

* **解决方案：**

  * 访问 Google Support 链接验证账号年龄；

  * 将 Chrome 设为默认浏览器并开启“Turn”模式（全局代理）；

  * 最彻底的方法是更换干净的 IP 节点并注册全新的 Google 账号。

**2. GPT 客户端“降智”与网络延迟**
近期普遍反馈 GPT 手机端和电脑端 App 响应极慢或变笨（降智），而网页版相对正常。

* **现象：** 客户端思考很久才回答，或直接报错。

* **OpenRouter 限制：** 使用 API 调用 GPT 模型时，OpenRouter 会识别 IP，部分官方服务商（如 OpenAI 直接提供）会拒绝服务，而第三方 Provider 则可能通过。

* **热点连接问题：** 电脑连手机热点无法使用 ChatGPT，建议手机和电脑同时开启代理工具。
