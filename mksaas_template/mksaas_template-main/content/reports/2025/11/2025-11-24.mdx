### 🚀 新品发布：不用写提示词的 AI 画图产品

**背景：** 小排老师发布了新产品 **Morisot.ai**，主打“在全世界教你写Prompt时，做一个不用Prompt的产品”。

* **产品特点：** 域名取自印象派女画家莫里索，延续了以艺术家命名的风格。产品旨在降低用户门槛，无需学习复杂的提示词工程。



### 🛠 技术实战：Cursor 开发与 GitHub 同步

**核心问题：** 多位新手开发者在使用 Cursor 写代码时，遇到代码无法同步到 GitHub 远程仓库的问题。

* **避坑指南：**

  * **区分 Commit 与 Push：** 在 Cursor 或 VS Code 中，“提交（Commit）”仅保存到本地仓库，“推送（Push）”才是上传到 GitHub 服务器。

  * **远程关联：** 如果是新项目，必须确保本地仓库已关联远程 URL（以 `.git` 结尾的地址）。

  * **操作建议：** 新手若搞不定图形界面（UI）操作，可直接要求 Cursor 的 AI 助理“帮我提交并推送到远程”，或尝试使用 GitHub Desktop 客户端，可视化操作更直观。

* **数据库操作：** 关于 Cursor 能否直接操作 Supabase 建表，建议复杂表结构直接运行 SQL 脚本，比单纯依赖 AI 对话更稳定。

### 💡 产品心法：RAG 开发陷入“过度设计”怎么办？

**背景：** 有圈友在开发“AI 销售助手”时，陷入技术泥潭（表结构复杂、埋点多、RAG 链路难懂），导致进度停滞。

* **关键建议：**

  * **坚守 MVP 原则：** 不要试图一次性做大而全的系统。只做一个核心功能，跑通了再做下一个。

  * **巧用工具：** 对于不熟悉 RAG（检索增强生成）原理的新手，建议先用 **Coze (扣子)** 搭建知识库并生成 API，Coze 已封装好 RAG 流程，能大幅降低开发难度。

  * **内功修练：** AI 虽然能写代码，但开发者仍需具备基础的理解能力（如读懂表结构），否则无法判断 AI 产出的对错。

### ⚖️ 其他话题

* **App Store 审核：** 关于提交审核被拒，建议直接将苹果的拒信或合规文档发给 AI，让 AI 检查代码并生成申诉邮件。通常按要求修改后回复邮件即可通过，无需硬等 3 个月。

* **工作流平台雏形：** 群友分享了一个自建的 Workflow 接单平台雏形，旨在连接甲方需求与乙方服务商，目前处于 1.0 完善阶段。

### 📝 总结

1. **网络不通先别慌**，去官网更新订阅地址通常能解决 90% 的问题。

2. **产品开发切忌贪大**，新手做 AI 应用先用 Coze 等低代码平台跑通 MVP，再考虑手写复杂后端。

3. **Cursor 虽然强大**，但理解 Git 的基础概念（本地 vs 远程）依然是必修课。

4. **小排新产品上线**，值得关注其“去 Prompt 化”的交互设计思路。
