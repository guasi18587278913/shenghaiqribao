### 🚀 技术焦点：Clawdbot/Moltbot 的低成本部署与风控

围绕刘小排分享的文章，群友们深入探讨了如何以更低的成本部署 AI 编程机器人（Clawdbot/Moltbot）。

* **部署方案对比（云端 vs 本地）：**

  * **成本差异：** 相比购买 Mac Mini 进行本地部署，大家发现利用腾讯云等服务商的**海外节点**（如日本节点）部署成本更低，被戏称为“省下了 4000 元硬件费”。

  * **形象比喻：** 有群友将云端部署比喻为“给实习生单独配一台电脑”，而本地部署则是“让实习生和你共用一台电脑”。

  * **优劣势分析：**

    * **云端优势：** 成本低、易扩展、且不占用本地资源，已有大量公开的 Clawdbot 部署在云端。

    * **本地优势：** 数据安全性更高（尤其涉及个人隐私数据时），且无需担心网络延迟。

* **账号风控与防封号：**

  * 针对 Claude 账号容易被封的问题，实战经验建议使用 **Codex 订阅** 或 **第三方 API**，避免直接连接个人主账号，因为一旦封号可能无法退款且损失历史数据。

  * Telegram 注册问题：国内 +86 号码注册困难，建议使用 APP 端或更换英美/香港手机号。

* **飞书/Lark 集成：**

  * 配置好 SSH 免密登录后，可以让 Claude Code 协助完成环境安装，进而接入飞书生态。

  * 虽然配置文档、日历、Wiki 等权限较耗时，但在飞书内部直接调用 AI Agent 对团队协作效率提升明显。

### 🛠️ 实操指南：Chrome 内置 Gemini 开启避坑

谷歌浏览器（Chrome）最新版内置的 Gemini Nano 模型引发了一波尝鲜热潮，但许多用户升级后并未看到图标，群内总结了有效的开启姿势：

* **开启条件：**

  1. 更新 Chrome 至最新版本；

  2. 网络环境需全局切换至**美国节点**；

  3) 浏览器语言设置建议改为**英文**。

* **替代方案：** 若常规设置无效，可使用开源脚本强制开启（见下方资源），或尝试在 chrome://flags 中调整设置。

* **能力边界：** 目前该功能主要表现为侧边栏对话（类似 Dia），**暂不支持**类似 RPA 的全网页操控（Browser Use），想用它自动点击网页按钮目前还行不通。

### 💡 效率工具与跨境电商

* **小红书调研新路子：** 有群友分享使用 **xiaohongshu-mcp**（Model Context Protocol）配合 Claude 进行市场调研，亲测稳定运行一个月，能有效抓取和分析小红书数据。

### 📝 总结与结论

1. **部署共识：** 对于只想尝鲜或轻度使用的用户，购买海外 VPS 部署 AI Agent 是比购买 Mac Mini 更具性价比的选择。

2. **风控提醒：** 玩 Claude 相关工具务必做好账号隔离，优先使用 API 或 Codex 订阅，保护主账号安全。

3) **技术现状：** 浏览器内置 AI（Gemini in Chrome）目前主要用于辅助对话，尚不具备复杂的网页操作（RPA）能力。

4) **新趋势：** MCP（模型上下文协议）在特定场景（如小红书调研）的应用开始落地，值得关注。
