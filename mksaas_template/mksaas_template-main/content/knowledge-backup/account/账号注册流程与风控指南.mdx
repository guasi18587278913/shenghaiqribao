# 账号注册与安全

## Claude账号注册与安全

适用对象：计划直接使用官方 Claude Web、Claude Code、Claude App 的成员。该服务对环境与身份的风控最严，务必在动手前把网络、手机号、支付方式全部准备好。

### 风险与前置条件
- **封号几乎不可逆**：志愿者欣霖、SuperEric 多次提醒，一旦环境异常会立即封禁，只能重新注册。
- **纯净静态住宅 IP 是底线**：注册前先用 `https://ping0.cc/`、`https://scamalytics.com/` 检测风险值；Medium/High Risk 的节点一律换掉。
- **环境隔离**：同一套 IP、浏览器指纹、操作系统只绑定一个账号。不同账号要在虚拟机/不同用户下隔离，避免串号。

### 手机号与账号准备
- 国内手机号在 Claude 场景下几乎必封，需提前准备海外号码。
- 节点国家要与手机号国家匹配；未匹配会触发验证或封号。
- 注册 Claude 前请先准备好 Gmail/Google Play 生态账号，后续订阅也要用到。

#### 常用手机号方案
- **giffgaff 英国 SIM（最稳妥）**
  - 购买：拼多多 6–20 元、闲鱼约 20 元，记得选择空卡自行激活。
  - 激活：首次充值 10 英镑，带邀请码到账 15–25 英镑，支付用纯 VISA/Mastercard。
  - 保号：每 180 天跑一次流量或发短信即可；10 英镑余额理论上能用 5 年以上。
  - 功能：可接收短信、拨打电话、支持 WiFi Calling。体验贴近真实手机号，适合长期使用。
- **摩尔多瓦 eSIM**
  - 成本：40 列伊（约 16 元），无需实体卡、无需实名。
  - 操作：选择 +373 号段 → 购买 40 列伊套餐 → 在线开通 → 即可收短信。
  - 注意：需要自己在官网开通，不要找中间商代办以防诈骗。
- **美国 Purple Card（紫卡）**
  - 成本：约 3 美元/月，有月租。
  - 优势：为美国节点准备的稳定手机号，可搭配美区网络环境。
  - 注意：需按月续费，记得设置提醒。
- **接码平台（一次性验证）**
  - 代表：PingMe、freephone.io、SMS-Activate。
  - 成本：几元到十几元一次；号码可能被反复使用，适合练手或补救。
  - 操作要点：国家与梯子一致；验证码未到就换号；成功率非 100%，要准备多试几次。
- **成品号 / 中转号**
  - 淘宝、闲鱼有已经注册好的 Claude 成品号；群友反馈有稳定用几个月的案例。
  - 购买后第一时间改密码、绑定自己的邮箱，并配合纯净 IP 使用。
  - 也可使用春秋哥中转（`https://aicodewith.com/?invitation=UFAV6EW`）、AnyRouter（`https://anyrouter.top/register?aff=KOmR`，每日签到送 25 美金额度）等可靠中转。

#### 环境组合成功案例
- @Ethan：使用 V2free（中转）+ Kookeey 静态住宅 IP + Clash Verge，直接用 Google 账号注册，无需手机号验证。
- 多位同学反馈：选择美国住宅 IP + 当地实体 SIM，注册成功率最高；香港节点常被拒。

### 正式注册流程
1. 连接风险值 &lt;30 的纯净节点，确认浏览器、终端都走同一代理。
2. 用准备好的 Gmail 访问 Claude 官网，按提示填写信息。
3. 进行手机号验证（或使用下方绕过方案）；完成后保存欢迎邮件。
4. 首次登录后，在同一环境下完成基础使用测试，再进入绑定支付或安装 Claude Code 等步骤。

### 电话验证绕过方案（助教壹塵分享）
1. 在 Google Play 安装 Claude App，并用同一个 Google 账号登录。
2. Google 侧完成 OAuth 授权后返回 App，即可绕过电话号码验证。
3. 仍需保证使用纯净 IP；若要订阅，在 Google Play 中绑定海外信用卡即可。

### 常见问题排查
- **401 Invalid bearer token**  
  场景：网页可用，终端报错。  
  处理：让 Claude 自查配置文件，删除旧的 `anthropic_token` 路径；若仍失败，换到日本等成功率更高的节点。
- **频繁封号 / 登录异常**  
  处理：被封后无法解封，只能重新注册并更换谷歌账号。保持节点稳定、避免短时间内大量 API 调用。
- **无法通过验证**  
  先检查节点是否为住宅 IP；必要时换客户端、清理浏览器缓存，再重新尝试。

### 替代与过渡方案
- 纯净环境尚未搞定时，可使用国内套壳（例如 2233）体验功能，优先保持学习节奏。
- 初学阶段可先在 Cursor 内使用 Claude 模型，等产品成型或需要高级功能时再申请官方账号。

## Google账号注册与防封

Google 账号是出海生态的“万能钥匙”，几乎所有平台（GitHub、ChatGPT、Claude、Bolt、V0、Lovable、Cursor、API 服务等）都优先支持 Google 登录。

### 注册准备
- 节点优先级：美国住宅 > 香港住宅 > 其他地区。注册前用 `ipjiance.com`、`ipinfo.io`、`ping0.cc`、`ip111.cn` 反复确认风险值 &lt;30。
- 设备建议：手机端（Chrome + 香港节点）成功率最高；其次是电脑端挂美国节点。
- 基本流程：打开 `google.com` → 创建账号 → 填写个人信息 → 绑定备用邮箱和手机号。

### 手机验证与错误处理
- 提示“此号码无法验证”时：
  1. 更换更干净的节点或改用 4G/5G 网络。
  2. 换到手机端重试。
  3. 间隔一天再试，或使用接码平台（`https://freephone.io/private-numbers` 等）。
- 国内手机号大概率可以验证，直接填 +86；被拒就换节点或换号码。
- 遇到可跳过验证的选项时可以暂时跳过，等账号稳定后再补充。
- 志愿者 Eric 的经验：不要在短时间内反复提交验证请求，放几天再试成功率更高。

### 养号与防封策略
1. **注册当天**：立即开启 2FA、添加辅助邮箱、设置头像与个人资料。
2. **养号期（至少 2 个月）**：
   - 固定节点登录，不频繁换 IP。
   - 用真实需求驱动账号：浏览 YouTube、收发邮件、搜索信息。
   - 注册第三方服务时，两个项目之间至少间隔 1–2 天。
3. **稳定期**：日常使用维持 1–2 个固定节点即可；老账号偶尔切换问题不大，新账号尽量避免。

### 常见风险与应对
- 账号被停用：按 Google 官方申诉流程填写，说明为正常个人使用，1–2 个工作日内通常会恢复。
- 需要密集注册大量服务：可购买 2023 年前的老账号。购买后立刻修改密码、辅助邮箱、手机号，同时保持纯净 IP。

## GitHub账号配置

### 注册与登录策略
- 直接用 Google 账号一键登录 GitHub 最省心；如需保留独立身份，可用同一个邮箱再注册 GitHub 原生账号。
- 日常浏览、克隆仓库用 Google 登录即可；涉及推送代码、配置 SSH、CI/CD 时，建议保留 GitHub 原生账号，方便生成 PAT/Token。
- 注册时地址选择中国没有影响，可根据实际情况填写。

### 手机验证
- GitHub 偶尔要求短信验证，国内手机号多数情况下可用。
- 遇到拦截：换更干净的节点，或使用接码平台（`https://freephone.io/` 等）。部分付费能力可能要求海外号码，提前准备备用计划。

### 账号体系建议
- 尽量使用统一邮箱或 Google 账号去绑定 Cursor、Bolt、Vercel 等平台，避免权限错乱。
- 让浏览器记住登录信息，便于在多个 GitHub 身份间切换。

## 其他工具与平台账号

### Google Firebase Studio（免费替代方案）
- 链接：`https://studio.firebase.google.com/`，常清悟静推荐。
- 优势：内置 Gemini，聊天体验好，基本免费。
- 劣势：项目源码需要手动导出；功能不如 Bolt 完整。

## 综合避坑清单

### 账号与配置层面
- 纯前端项目要注意 API Key 的调用额度，避免密钥泄露后被恶意刷请求。（剑廿三）
- 使用接码平台时，务必保证号码国家与科学上网节点一致。（前行教练）
- GitHub 对异常提交有封号风险，提交前确认内容合规。（前行）
- Cursor 教育账号一旦开启 Max 模式会立刻被封，请勿尝试。（助教小明）
- Windows 环境的网络与开发配置复杂度是 Mac 的 2–3 倍，要预留排障时间。（银河星游）
- 华为手机（鸿蒙）无法直接使用 Google Play，需要准备其他设备或方案。（群友共识）
- API Key 只显示一次，生成后立刻保存；验证码邮件可能进入垃圾箱，要及时查看。

### 工作方式与产品推进
- **大坑 1：想着做完美产品才上线**  
  解决：给自己设定 Deadline，坚持 “够用就行” 的 MVP 思路。兔老师经验：真实、可用比精致 PPT 更打动评委。
- **大坑 2：卷巨头赛道**  
  代表坑：想复制 DIA 浏览器、做通用对话机器人。建议聚焦自身行业经验与垂直需求，做巨头不愿做的小场景。
- **大坑 3：过早优化**  
  错误心态：一开始就考虑国际化、架构、商业模式。正确顺序是先验证需求，再做最小可用产品，最后考虑规模化。

保持以上要点，可以显著提高账号保活率，也能避免在学习和实操阶段浪费时间。若未来需要扩展到更多平台账号，可在同一目录下继续补充相应章节。***
