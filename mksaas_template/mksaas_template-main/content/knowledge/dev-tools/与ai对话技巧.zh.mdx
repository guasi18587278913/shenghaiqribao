## 基础到进阶

<Callout type="warning">
**描述要具体**：不要只说"报错了"，而是提供操作步骤、完整日志、环境信息。

例如："Windows11，Node18.19.0，执行 `npm run dev` 后报 `Cannot find module 'react'`，我刚创建完项目。"
</Callout>

- **投喂上下文**：指出文件名、路径、预期效果、参考样式
- **分步迭代**：生成初版 → 自测 → 告诉 AI "哪里不好" → 继续优化

---

## 多模型互搏

- GPT 擅长生成原型，Claude Code 善于逻辑与重构。难题时让两个模型"互相审查"
- DIA/Comet 浏览器适合截图+提问；V0/Bolt 报错可直接丢给 DIA，或者"复制代码+截图给 GPT"让它诊断

---

## 关键要点

1. 明确目标（要什么页面/功能/交互）
2. 提供足够细节（文件、位置、数据、颜色）
3. 出错时附完整日志/截图

<Callout type="warning">
逐步逼近成品，不要一次塞满需求。
</Callout>

<Callout type="info">
用日志和代码块承载敏感信息时，记得上线前清理。
</Callout>
