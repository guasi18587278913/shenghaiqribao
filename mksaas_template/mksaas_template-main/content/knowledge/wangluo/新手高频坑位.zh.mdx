| 坑点 | 表现 | 正确做法 |
| --- | --- | --- |
| 纠结设备 | 一直研究买什么电脑 | 先用手头设备，需求起来再换（Eric） |
| 网络没配好就注册 | Claude 秒封、账号受限 | 梯子 → 测 IP → Low Risk 再动手（SuperEric） |
| 一次堆太多功能 | 功能多但 bug 多无法回退 | 先做最小可行功能，Git 分阶段提交（前行） |
| 订阅一堆会员 | GPT、Bolt、翻译都买付费 | 必选：GPT Plus；Cursor/Claude 视需求；沉浸式翻译免费版够用 |
| 忘记基础操作 | 不开 TUN 就说 Cursor 连不上、终端不在项目目录 | 用 checklist 自查：TUN、目录、依赖、日志 |

<Callout type="warning">
| 安全意识弱 | API Key 写死在前端、群里贴敏感截图 | 一律放 `.env`，加 `.gitignore`，日志里不要打印 key |
</Callout>

<Callout type="info">
| 期望 AI 一次写完美代码 | 结果变成“AI 不行” | 把需求拆小，迭代式提示 + 详细日志 |
</Callout>
