### 🔧 账号与配置层面

<Callout type="warning">
- 纯前端项目要注意 API Key 的调用额度，避免密钥泄露后被恶意刷请求。（剑廿三）
</Callout>
- 使用接码平台时，务必保证号码国家与科学上网节点一致。（前行教练）
- GitHub 对异常提交有封号风险，提交前确认内容合规。（前行）
- Cursor 教育账号一旦开启 Max 模式会立刻被封，请勿尝试。（助教小明）
- Windows 环境的网络与开发配置复杂度是 Mac 的 2–3 倍，要预留排障时间。（银河星游）
- 华为手机（鸿蒙）无法直接使用 Google Play，需要准备其他设备或方案。（群友共识）
- API Key 只显示一次，生成后立刻保存；验证码邮件可能进入垃圾箱，要及时查看。

### 工作方式与产品推进

<Callout type="info">
**大坑 1：想着做完美产品才上线**
</Callout>

解决：给自己设定 Deadline，坚持 "够用就行" 的 MVP 思路。兔老师经验：真实、可用比精致 PPT 更打动评委。

**大坑 2：卷巨头赛道**

代表坑：想复制 DIA 浏览器、做通用对话机器人。建议聚焦自身行业经验与垂直需求，做巨头不愿做的小场景。

**大坑 3：过早优化**

错误心态：一开始就考虑国际化、架构、商业模式。正确顺序是先验证需求，再做最小可用产品，最后考虑规模化。

<Callout type="warning">
保持以上要点，可以显著提高账号保活率，也能避免在学习和实操阶段浪费时间。若未来需要扩展到更多平台账号，可在同一目录下继续补充相应章节。***
</Callout>
