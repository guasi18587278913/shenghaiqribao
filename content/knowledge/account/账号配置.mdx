# github-账号配置

### 注册与登录策略
- 直接用 Google 账号一键登录 GitHub 最省心；如需保留独立身份，可用同一个邮箱再注册 GitHub 原生账号。

> **Note (info):** - 日常浏览、克隆仓库用 Google 登录即可；涉及推送代码、配置 SSH、CI/CD 时，建议保留 GitHub 原生账号，方便生成 PAT/Token。
- 注册时地址选择中国没有影响，可根据实际情况填写。

### 手机验证
- GitHub 偶尔要求短信验证，国内手机号多数情况下可用。
- 遇到拦截：换更干净的节点，或使用接码平台（`[freephone.io](https://freephone.io/`) 等）。部分付费能力可能要求海外号码，提前准备备用计划。

> **Note (info):** ### 账号体系建议

> **Note (warning):** - 尽量使用统一邮箱或 Google 账号去绑定 Cursor、Bolt、Vercel 等平台，避免权限错乱。
- 让浏览器记住登录信息，便于在多个 GitHub 身份间切换。
