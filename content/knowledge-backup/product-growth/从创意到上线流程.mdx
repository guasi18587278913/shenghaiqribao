# 从创意到上线流程

### 3.1 阶段划分（基于“从创意到落地的完整流程”）
1. **Idea 验证（1–2 周）**
   - 需求调研：TikTok、Reddit、Product Hunt、Flowith 等工具寻找真实讨论。
   - 竞品分析：列功能、价格、用户评价、流量来源，找“软柿子”。
   - 付费潜力判断：看竞品是否有付费功能、是否出现在付费榜单、用户是否已经为类似问题付钱。
   - 输出 MVP 功能清单：只保留 20% 的关键功能，目标是 1–2 周内上线。

2. **MVP 开发（2–4 周）**
   - 工具组合：V0/Bolt 快速出初版 → Cursor 精细迭代 → 尽量复用现成 API，减少自建后端。
   - 节奏建议（兔老师）：早期每天 2–3 小时；中期 4–6 小时集中实现；后期 1–2 小时维护优化。卡住就休息，换个角度再问 AI。

3. **测试与部署（约 1 周）**
   - 功能回归：核心流程、异常、支付链路。
   - 部署事项：域名、SSL、可选 CDN，加上线前的 Checklist。
   - 营销素材：落地页文案、演示视频/动图、社媒账号。

4. **推广获客（持续迭代）**
   - TikTok / 小红书：记录真实开发过程、发布痛点解决案例。
   - Product Hunt：准备发布日，争取榜单曝光。
   - Reddit：养号合规后进入细分社区分享（有被封风险）。
   - 行业论坛 / Newsletter / 个人 IP：持续输出复盘内容，建立信任。

### 3.2 竞品“抄法”与差异化
- 不是照抄，而是“选软柿子 + 微创新”：
  - 找时间不长、技术门槛不高、关键词竞争小的产品。
  - 结合本地化/垂直化场景（如只做学术引用管理、只做特定行业计算器）。
  - Samuel/一人公司的案例：复刻 3 个 APP 并做改进，月入 3.5 万。
