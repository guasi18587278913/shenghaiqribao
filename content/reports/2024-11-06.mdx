## 1. 网络环境配置

### 1.1 节点纯净度检测

**问题场景**：注册Claude等服务时提示需要验证，可能是节点不干净

**解决方案**：
- 使用 https://ping0.cc/ 检测节点纯净度
- **纯净度标准**：
  - 0-25：低风险（优秀）
  - 25-40：中风险（一般可用）
  - 40+：高风险（不建议使用）

**群友实测**：
```
海天-遂宁：28分的风险值
助教-小明：算好的了
曹玉成：低风险就可以，不爆黄应该问题不大
```

### 1.2 DIA浏览器配置问题

**推荐配置**（舒未读分享）：
```
美国节点 + Mac（M芯片）
节点纯净度检测通过后使用
```

**替代方案**（群友推荐）：
- Windows用户：豆包浏览器、QQ浏览器、ima浏览器
- Chrome插件：Side Panel for Gemini（搜索安装即可）

### 1.3 网络代理模式选择

**问题**：全局模式vs绕过大陆模式的选择

**建议**：
- 如果绕过大陆模式有网站打不开，可以手动添加域名到规则列表
- 不建议长期使用全局模式
- 自行搜索如何修改规则列表，添加需要访问的域名

## 6. 开发环境配置

### 6.1 Next.js项目创建

**完整流程**（群友实践）：

**步骤1：检查Node.js**：
```bash
node -v
npm -v
# 注意中间有空格
```

**步骤2：切换npm源**（周尹提供）：
```bash
npm config set registry https://registry.npmmirror.com
```

**步骤3：创建项目**：
```bash
npx create-next-app@latest my-app
# 会提示是否安装，输入y确认
# 注意不要在提示时输入其他命令
```

**步骤4：进入项目并启动**：
```bash
cd my-app
npm run dev
# 成功后会显示：http://localhost:3000
```

**步骤5：访问项目**：
```
复制 http://localhost:3000 到浏览器访问
看到页面说明配置成功
```

**常见错误排查**：
```
错误：Could not read package.json
原因：
1. package.json文件未下载（网络问题）
2. 路径定位错误
解决：
- 先运行npm config set registry命令
- 确保在正确的目录下
- 节点能正常访问
```
