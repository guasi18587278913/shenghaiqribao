## 💡 二、AI编程工具完全指南

### 2.1 工具体系认知

**核心理念**
小排老师的教学体系中，工具选择遵循"新手友好→专业强大"的路径。不是一开始就用最强的工具，而是循序渐进。

**工具分层**
```
学习阶段        工具选择              原因
─────────────────────────────────────────
入门期(1-30天)   Cursor              界面友好，容错率高
过渡期(30天+)    Claude Code/Codex   专业工具，性能强大
备选方案         豆包TRAE/GLM        国产，价格友好
```

### 2.2 Cursor - 新手首选

**为什么选Cursor**
- 群友原话："cursor现在是新手最友好的AI编程学习工具"
- 界面直观，上手快
- 错误提示清晰，适合学习

**如何获取Cursor**
1. **推荐方式**：淘宝/闲鱼购买一个月
   - 价格：约30-100元/月
   - 有人买的是"试用账号+换号器"，测试可用Claude 4.5
   - 注意：会稍慢，但能完成写代码任务就行

2. **自己订阅**：不推荐
   - 因为Cursor在课程中只是前期过渡工具
   - 约30天后就要转向CC，自己订阅不划算

**Cursor使用注意事项**
- 内存泄漏问题：
  - 现象：好几个月不关机可能遇到内存爆掉
  - 解决：电脑重启即可，然后又可以撑很久
  - Mac M1 16GB用户：约一周会爆内存
  - 小排老师也遇到过这个问题，重启解决

- 实际体验：
  - 使用Gemini 2.5 flash解释和分析代码免费，效果很好
  - 有助教表示有四五个AI助手同时帮忙干活

### 2.3 Claude Code - 专业工具核心

**什么时候转Claude Code**
- 明确信号："当你发现cursor的弊端的时候，就该转CC了"
- 时间点：进度大概在课程的30天
- 不要急于一开始就用，先用Cursor打好基础

**Claude Code的优势**
- 专业级AI开发工具
- 写代码更准确，更能理解复杂需求
- 群友分享："claude 4.5比GLM-4.6更能遵循我的命令"
- 在AI大会分享中，大部分指令调用都选择claude

**Claude Code两种使用方式**

方式一：官方插件（VSCode）
- 安装官方Claude Code插件
- 需要订阅Claude账号
- 可能遇到网络问题（需要美国节点）

方式二：终端使用
- 直接在终端运行claude命令
- 助教建议：如果插件有问题，可以先用终端测试

**常见问题与解决**

问题1：插件报错 "SyntaxError: Unexpected token 'C'"
- 原因：官方插件的bug
- 解决方法：新开一个聊天窗口
- 如果还不行：回退到旧版本（2.0.08或2.0.10）
- 有的人升级到2.0.17也能正常使用

问题2：强制登录问题
```bash
# 解决方案：新建配置文件
# 路径：~/.claude/config.json
# 内容：
{
  "primaryApiKey": "fox"
}
```

问题3：网络连接问题
- 必须使用美国节点
- 香港IP不支持
- 浏览器能访问但插件不行：通常是IP问题
- 检查代理设置是否正确

问题4：WSL环境配置难题
- 群友经验："WSL很难用的，我配置了代理搞了两三天"
- 最终解决：换用Clash的TUN模式
- 但TUN模式可能只有香港IP，claude code又不支持
- **建议：直接换Mac，不要在WSL上浪费时间**

**Claude Code Yolo版本**
- 这是圈友逆向升级的版本
- 某些情况下比官方版本更稳定
- 如果官方版本有问题可以尝试

### 2.4 Codex - 另一个专业选择

**Codex特点**
- OpenAI出品，与Claude Code并列的专业工具
- 可以用GPT的team账号
- 价格相对便宜（team账号闲鱼10-30元/月）

**Codex使用方式**
- 推荐：使用Codex官方插件（比较友好）
- 也可以使用Codex CLI

**Codex vs Claude Code**
群友讨论：
- 有人问："codex好用吗？用了几次好像比cc差"
- 但也有人觉得够用
- **建议：两个都试试，找到适合自己的**

**Codex的隐藏字问题**
- 有群友遇到Codex显示时出现隐藏字的情况
- 可能是显示bug，不影响实际使用

### 2.5 国产替代方案

**豆包TRAE**
- 特点：国产，网络稳定
- 价格：相对便宜
- 适用场景：
  - 网络环境不好的情况
  - 预算有限
  - 备用方案
- 群友反馈："trae,个人觉得很好用，买了一年的"

**GLM-4.6（智谱清言）**

详细定价：
- Lite版：价格便宜
- Pro版：约600元/3个月
- 计费方式：按请求次数，不是token
- 拼好模可以打折，更便宜

性能对比：
- 官方说法："不如claude 4.5，但是比claude 4强一些"
- 群友实测："claude 4.5比GLM-4.6更能遵循我的命令"
- 适合场景："可以先买Lite，不够用再升Pro"

使用建议：
- output改成explain模式解释代码很好用
- 地板价，预算有限的首选
- 但写代码还是不放心GLM-4.6，关键项目建议用claude 4.5

订阅链接：
- 拼好模链接：https://www.bigmodel.cn/claude-code?ic=T6UBWALTWW
- 使用这个链接可以省钱

**其他工具**

Augment Code：
- 也是基于Claude Sonnet 4.5
- 特点：把Sequential thinking直接嵌入使用
- 群友反馈："太高效了，有些代码Claude Code写不出来，Augment就能写出来"
- 工具和工具之间相差很大，即使用相同的模型

Droid：
- 提供免费额度
- 邀请码：https://app.factory.ai/r/T429S35Q
- 邀请双方各得40刀
- 曾在Terminal-Bench基准测试中拿下58.75%成绩
- 但有群友反馈："刚开的会员，用了30分钟就拉了"（可能宕机）

CodeBuddy：
- 也是备选方案之一
- 具体使用情况群里讨论较少

### 2.6 工具订阅的经济账

**新手阶段（1-30天）**
- Cursor淘宝：30-100元/月
- 够用，成本低

**进阶阶段（30天+）**
选择一：自己订阅Claude
- Claude Pro：20美元/月
- Claude Code支持，但用量有限

选择二：中转服务
- 春秋大哥的：按使用量计费
- 优势：不用担心封号，稳定

选择三：GPT Team + Codex
- GPT Team闲鱼：10-30元/月
- 用量比Plus多
- 群友分享："比plus用量更多，我知道之后再也不买plus了"

**月度成本对比**
```
方案                  费用          稳定性    适用场景
───────────────────────────────────────────────────
Cursor淘宝           30-100元       中        新手入门
中转(春秋)           100-300元      高        长期使用
Claude官方订阅       200美元        中        重度使用
GPT Team(闲鱼)       10-30元        中        经济方案
GLM Pro              200元/月       高        国内备用
```

**省钱技巧**
1. 新手先用Cursor，不要一开始就买贵的
2. 中转服务对新手来说日额度大概率用不完
3. 多个工具结合使用，根据任务选择
4. 备用多个账号，一个被封立即切换

## 🔧 三、工具订阅完全攻略

### 3.1 订阅策略总览

**核心思路**
群友讨论得出的共识是：
1. 新手推荐用中转，日额度大概率用不完
2. 能自己正经支付最稳定（但成本高）
3. 多准备几个方案，互相backup

**风险提示**
- CC（Claude Code）很容易封号，最近大封
- 群友分享："最近大部分人都被封了，我用礼品卡充的也被封了"
- 即使用美国银行卡付款也会被封账户
- **核心应对策略**：听教练推荐的直接找代理买，或用中转

### 3.2 方案一：中转服务（最推荐）

**春秋大哥的AI Code With**

基本信息：
- 官网：https://aicodewith.com/?invitation=UFAV6EW
- 这是小排老师推荐的，"有保障"
- 支持：Claude Code、Codex，后续会接入Gemini 3
- 95折优惠（通过推荐链接）

优势分析：
1. 不用担心封号问题
2. 新手日额度大概率花不完
3. 稳定性高，小排老师亲自推荐
4. 省去自己配置环境的麻烦

使用反馈：
- 群友："可以用我的推荐链接呀，我天天群里给大家推荐春秋大哥的"
- "自从买了春秋大哥的我都不冲claude了"

**中转服务的原理**
- 提供API接口
- 你不直接面对官方，避免封号风险
- 按使用量计费，用多少花多少

**适用人群**
- 新手（强烈推荐）
- 不想折腾环境配置的人
- 害怕被封号的人
- 需要稳定服务的人

### 3.3 方案二：Apple礼品卡订阅

**尼日利亚苹果ID方案**

完整流程：

第一步：准备Apple ID
- 注册一个尼日利亚地区的Apple ID
- 或者切换现有ID到尼日利亚区（有风险）

第二步：购买礼品卡
- 在淘宝搜索"尼日利亚苹果礼品卡"
- 价格：Claude Max约1100+人民币
- Claude Pro约140-160元（20美元等值）

第三步：充值
- 打开App Store，切换到尼日利亚账号
- 兑换礼品卡
- **重要**：不要一次性充值太多

第四步：订阅
- 在iPhone上打开Claude App
- 选择订阅（会用礼品卡余额支付）
- 走苹果内购通道

**价格对比**
- Claude Pro：约140-160元人民币（尼日利亚区）
- Claude Max：1100+人民币（尼日利亚区）
- 对比官网：20美元和200美元
- 节省幅度：不大，主要是解决支付问题

**风险与注意事项**

风险一：苹果ID被封
- 现象："不过礼品卡充值尼日利亚苹果id有封苹果id的情况，我已经碰到了"
- 原因：密集充值触发风控
- 表现：
  - 限制购买（不是直接封号）
  - 但解不了，联系客服也没用
  - 有人限制后找客服解除了，但也有人解不了

风险二：变相涨价
- Claude现在也20美元了，尼日利亚区优惠没了
- GPT土耳其还有优惠

**优化方案**
- 不要一口气充20W（有人这么干结果被限制）
- 分批充值，每次少充一点
- 被限制可以尝试联系苹果客服解除
- 准备多个Apple ID备用
- 用信用卡代替礼品卡："尼日利亚苹果id+信用卡，貌似没问题"

**验证方法**
群友给出的可用性确认：
- "唬人的，我昨天刚用礼品卡开了20x"（意思是20美元版本）
- "尼卡我充cc 9月中试了可以，要申诉解除限制"
- 表示目前（2024年10月）还是可用的

**替代区域**
土耳其Apple ID：
- 有群友用土卡订Code X成功
- 6折价格
- CC没试过，但理论可行

### 3.4 方案三：虚拟卡方案

**VCC（Virtual Credit Card）完整流程**

第一步：购买USDT
- 在火卡（或其他平台）购买USDT
- 这一步相对简单，火卡支持支付宝

第二步：注册VCC平台
- 有多个VCC平台可选
- 生财有帖子详细介绍（可搜索"野卡跑路后 Claude/ChatGPT充值方法"）

第三步：充值VCC
- 用USDT充值到VCC账户
- 手续费：
  - 充值20刀：1.5刀手续费
  - 充值200刀：4.2刀手续费
  - 充值越多，手续费率越低

第四步：开卡
- 在VCC平台申请虚拟卡
- 美国、欧洲等地区可选

第五步：订阅Claude
- 用虚拟卡直接订阅Claude官方
- 和普通信用卡使用方式一样

**注意事项**
- 野卡（WildCard）已经跑路："被有关部门立案调查了，资金冻结"
- 有人用野卡还欠300U没退回来
- Bybit虚拟卡前两个月好用，这个月开始疯狂封号
- **结论**：虚拟卡方案不稳定，风险高

**为什么容易被封**
- Claude系统检测到虚拟卡
- IP和支付地址不匹配
- 大量虚拟卡来自同一个卡段
- 群友经验："我现在都麻了"

### 3.5 方案四：香港银行卡（最稳定但门槛高）

**汇丰One线上申请方案**

前置条件：
- 需要去一趟香港（哪怕只是过境）
- 需要港澳通行证

完整流程：

第一步：入境香港
- 拿着港澳通行证过关
- 只要有出入境记录即可
- 之前传言需要待满三天已经修复："过去38分钟就有了"

第二步：线上申请
- 在香港境内打开汇丰银行APP
- 申请汇丰One账户
- 时间：约半个小时

第三步：等待卡片
- 平邮到内地地址
- 约7天到手

第四步：激活使用
- 收到卡后激活
- 可以用来订阅Claude、GPT等

**注意事项**
- 必须人在香港境内才能线上申请
- 汇丰One是借记卡（Debit Card）
- 借记卡无法办信用卡服务
- 但足够用来订阅AI服务了

**无需去香港的方案**
- 在内地汇丰存50万人民币
- 可以顺带办理香港汇丰账户
- 需要存放几个月
- 成本高，不推荐普通人

**众安银行方案**
- 也是香港银行
- 需要安卓手机
- 绑定到Google Pay使用
- 可以用来订阅服务

**实际使用**
群友晒图：每月自动扣款，很稳定
- 不用担心虚拟卡被封
- 不用每月手动充值
- 真实银行卡，被封概率极低

**Stripe注册问题**
有群友反映：
- 香港公司注册下来了
- 但Stripe要求"香港人控制的"
- 解决方法：
  - 买英国手机卡（Giffgaf）收短信
  - 或者直接用香港银行卡注册个人Stripe账户
  - 个人账户申请快，几小时就能批

### 3.6 方案五：招商全币通（最简单）

**适用场景**
- 不想折腾虚拟卡
- 不方便去香港
- 想要正规途径

**办理流程**
1. 去招商银行网点
2. 申请全币通信用卡
3. 等待审批和制卡
4. 收到卡后激活

**优势**
- 正规银行卡
- 外币支付直接从额度扣
- 还款方便（自动购汇还人民币）
- 不容易被封

**劣势**
- 需要信用记录
- 有些人申请不通过
- 额度可能不够

### 3.7 GPT订阅特殊方案

**Team账号薅羊毛**

核心发现：
- GPT Team版本比Plus用量更多
- 价格只要10-30元/月（闲鱼）
- 群友分享："我知道之后再也不买plus了"

具体操作：

第一步：获取多个Gmail
方法：一个谷歌邮箱可以变出无数个
- 加点号：example@gmail.com → exam.ple@gmail.com
- 加加号：example@gmail.com → example+1@gmail.com
- 改后缀：@gmail.com → @googlemail.com
- 生财有教程可搜

第二步：注册多个GPT账号
- 用变体邮箱注册
- 实际都会收到同一个邮箱

第三步：购买Team账号
- 闲鱼搜索"GPT Team"
- 价格10-30元不等
- 购买后卖家会把你加入Team

第四步：切换使用
- 唯一不方便：登录新账户时其他窗口不能同时对话

## 💳 五、支付网关与公司注册实战

### 5.1 核心认知

**什么时候处理支付？**
助教明确指出：
- "这个属于课程很后期的内容，先不用急"
- "先开发网站，网站有一定日活后再申请支付"
- **策略**：先把产品做出来，有人用了，再接支付

**为什么不着急？**
1. 申请支付网关需要有网站
2. 网关会审核你的产品
3. 没产品申请不通过
4. 过早申请浪费时间

**应对策略**
- 新手阶段：专注学习和做产品
- 有了MVP：再研究支付
- 有了流量：申请支付网关
- 被拒或被封：立即切换备用网关

### 5.2 对中国大陆友好的支付网关

**三大主流**
助教推荐顺序：
1. **Creem**
2. **Paddle**
3. **Stripe**

**其他备选**
- LemonSqueezy
- PayPal（需要特殊处理）

**为什么需要多个？**
群友建议："多注册几个备用，被封了可以立即切换"
- Creem审核严
- Paddle相对宽松
- Stripe最稳但有门槛
- 多手准备很重要

### 5.3 Creem完全攻略

**基本信息**
- 对中国大陆最友好
- 中国护照即可注册
- 无需公司（个人就行）
- 但审核很严格

**申请前置条件**

必须有：
1. 一个已经上线的网站
2. 网站有实际功能
3. 网站内容合规

不能有：
1. 违规内容
2. 虚假用户证言/数据
3. AI生成违规内容
4. 蹭大品牌词（如veo、claude等）

**Creem的审核策略**

关键发现：
> "creem会故意用生成色情的那种prompt测试，你要拦截的"

这意味着：
- 审核人员会尝试用你的AI工具生成色情内容
- 如果能生成，审核不通过
- 必须做好内容过滤

**AI产品如何通过审核**

群友成功经验：
> "我是AI游戏站，几个支付网关都过了"

具体措施：
1. 添加prompt过滤
2. 拦截敏感词
3. 设置内容审核
4. 测试账号要准备好

**失败案例分析**

案例1：AI图片站被拒
> "我是AI图片站，申请没过。拦截了，还给了他们测试账号，再次申诉的，结果他们连账号登录都没登录过，就又把我拒了"

教训：
- 即使做了拦截也可能被拒
- Creem对AI图片站审核特别严
- 需要多次申诉

案例2：提供测试账号
- 审核时主动提供测试账号
- 让审核人员能实际测试
- 证明你的过滤机制有效

**申请流程**

第一步：准备材料
- 网站URL
- 网站功能说明
- 商业模式说明
- 个人身份信息（护照）

第二步：填写申请
- 登录Creem官网
- 填写申请表
- 详细说明产品

第三步：等待审核
- 通常几天到一周
- 可能要求补充材料
- 可能要求提供测试账号

第四步：被拒后申诉
- 说明已添加的过滤措施
- 提供测试证明
- 可以多次申诉

**通过后注意事项**
- 持续监控内容合规
- 用户生成违规内容会被追责
- 保持良好的商户信誉

- 优势：更正规、支持更多功能
- 适合：正规运营

**香港银行卡方案详解**

步骤1：办理香港银行卡

步骤2：注册Stripe个人账户
- 访问Stripe官网

步骤3：绑定银行卡

步骤4：等待审核
- 个人账户审核很快
- 群友反馈："几个小时就搞定了"

**英国公司方案详解**

## 💻 六、硬件配置完全选购指南

### 6.1 核心认知

**是否必须Mac？**
群聊中的讨论反映了现实：
- Dia浏览器只支持Mac M芯片
- Claude Code在Mac上配置更简单
- 成功案例中大部分人用Mac
- 但Windows也能学习，只是折腾更多

群友直言：
> "话说咱们有案例的学员该不会10个里有8个用macbook吧"

**投资逻辑**
- Mac很好用，10年前的老Mac依旧能用
- 长远打算，上个好的，不花两道钱
- 群友建议："先用现有设备学习，赚钱了再换"
- 争取赚回学费和Mac的成本

### 6.2 Mac Mini方案（高性价比）

**适用人群**
- 公司有显示器
- 家里有显示器
- 预算有限
- 不需要移动办公

**推荐配置**

基础款：
- 芯片：M4
- 内存：16GB（最低）
- 硬盘：512GB
- 价格：闲鱼约2800元（M4版本）

进阶款：
- 芯片：M4
- 内存：24GB或32GB（推荐）
- 硬盘：512GB或1TB
- 价格：官网或京东

**内存选择的深度讨论**

16GB体验：
> "我是16gb的mac mini m4，差不多一周就能给我干爆内存"

群友建议：
> "16GB内存有点不够用，最好还是24，32GB的"

结论：
- 16GB：能用，但会吃紧
- 24GB：推荐，体验好
- 32GB：理想，预算充足就上

**硬盘选择**

助教意见：
> "512足够了"

但也有不同声音：
> "512够，但也会吃紧，预算充足上1t，避免烦恼"

实际建议：
- 512GB：大部分场景够用
- 1TB：如果预算允许，更好
- AI编程不需要太大硬盘（代码不大，调用API不是本地）

**为什么有人不推荐Mini？**

群里有争议：
> 问："macmini可以吗"
> 答："不推荐啊"
> 问："为啥啊"

但也有大量公司开发用Mini的案例。

不推荐理由可能是：
1. 需要额外买显示器
2. 不能移动办公
3. 首次购买Mac建议笔记本（更全能）

推荐理由：
1. 性价比极高
2. 性能足够
3. 适合固定场所工作
4. M4芯片性能强大

**使用场景**
- 在公司：直接用公司显示器
- 在家：配个显示器
- 需要时："回家直接扛回来"

### 6.3 MacBook Air方案（够用）

**推荐配置**

基础款：
- 芯片：M3
- 内存：16GB
- 硬盘：512GB
- 特点：轻薄，续航长

进阶款：
- 芯片：M4
- 内存：24GB
- 硬盘：512GB
- 更好的体验

**为什么Air够用？**
> "大家这里99%的同学都是直接调API做AI coding，mac studio明显性能过剩，用macbook air都行"

关键认知：
- 不是本地运行大模型
- 是调用Claude/GPT的API
- 所以不需要顶级性能
- Air完全够用

**实际体验**
有群友反映24GB内存的Air会卡，但这是少数情况，可能是：
- 同时开太多应用
- Chrome标签页过多
- 内存泄漏（重启解决）

### 6.4 MacBook Pro方案（首选）

**为什么Pro？**
群友共识：
> "MacBook Pro肯定是首选"

理由：
1. 性能更强
2. 散热更好
3. 接口更多
4. 屏幕更大
5. 长期投资，一步到位

**推荐配置**

标准款：
- 芯片：M3 Pro或M4 Pro
- 内存：24GB起
- 硬盘：512GB或1TB
- 尺寸：14寸（便携）或16寸（爽）

高配款：
- 芯片：M4 Max
- 内存：48GB或更高
- 硬盘：1TB或2TB
- 适合：重度用户

**极致配置案例**

有群友晒出：
- M4 Max芯片
- 96GB内存（128GB统一内存）
- 2TB硬盘
- 价格：约4万元

群友反应：
> "6"、"过分了"、"这个要4万了吧"

实用性：
- 这配置可以跑本地大模型
- 可以本地装大模型
- 但对于调用API来说确实过剩
- 土豪专属

### 6.5 Intel Mac的困境

**现实问题**
- Dia不支持Intel Mac
- 新工具优先适配M芯片
- 性能差距大

**群友困扰**
> "非常不幸我就是最后一版的intel，舍不得陪伴我6年的mac"

> "我也是intel，17 pro。我是M芯片，提示是系统版本不支持，在升级系统中"

**应对策略**

策略1：继续用
- 学习阶段够用
- Dia可以不用（有替代品）
- 用豆包等国产工具
- "先学习，赚钱了再换"

策略2：考虑换新
- 如果工作强度大
- 如果公司配电脑
- 如果预算允许
- M4性价比高

策略3：Mini过渡
- Intel Mac留着做其他用途
- 买个M4 Mini（2800元）
- 用原来的显示器
- 最经济的过渡方案

### 6.6 Windows用户方案

**Windows能学吗？**
- 答案：能，但要折腾
- Cursor支持Windows
- Claude Code需要配置
- WSL不推荐（坑多）

**Windows的劣势**
1. Dia不支持
2. 环境配置复杂
3. 某些工具Mac优先
4. 成功案例多用Mac

**Windows的优势**
1. 已有设备，不用买
2. 游戏性能好
3. 硬件便宜
4. 扩展性强

**建议**
- 先用Windows学基础
- 确定要深入后再买Mac
- 或者公司/二手渠道获得Mac

### 6.7 硬件需求的本质

**不需要做的事**
- ❌不需要本地装大模型
- ❌不需要顶级GPU
- ❌不需要很大硬盘
- ❌不需要最新最贵的设备

**真正需要的**
- ✅稳定的设备
- ✅流畅的网络
- ✅舒适的屏幕
- ✅能跑VSCode/浏览器

**群友总结**
助教说明：
> "不需要本地装大模型"
> "512足够了"

因为：
- 我们调用API，不是本地跑模型
- 代码文件不大
- 主要占空间的是开发环境和工具
