## 1. Cursor使用完整指南

### 1.1 网络问题终极解决方案

**问题表现：**
- 模型显示"Not Available"
- 提示"服务器网络问题"
- 无法连接Claude/GPT模型

**完整解决步骤：**

#### 第一步：设置Network为HTTP/1.1
```
位置：Cursor设置 > Network
操作：将协议改为 HTTP/1.1
```

#### 第二步：开启TUN模式（关键！）
```
位置：魔法工具（如Clash Verge）> 设置 > TUN模式
重要性：这一步经常被忽略，但是核心解决方案
```
**群友经验：**
- @田超："换ying国节点可以了"
- @勿忘心安："魔法开tun模式"
- @半糖："开了tun模式就可以了"

#### 第三步：节点选择策略
```
推荐节点：美国 > 英国 > 日本
避免使用：香港、澳门、台湾节点
节点要求：纯净度至少28以上
```

**完整配置截图位置：**
- 设置界面：有群友分享的完整配置图
- 关键设置都在 `Cursor Settings` → `Network` → `Override Proxy Settings`

### 1.2 Cursor项目管理核心概念

**新手最容易犯的错误：在错误的目录运行命令**

**正确的项目启动流程：**

```bash
# 1. 先打开或创建项目文件夹（重要！）
# Cursor左上角 File → Open Folder → 选择项目文件夹

# 2. 确认终端在正确目录
# 查看终端路径，应该显示你的项目文件夹路径

# 3. 运行项目
npm run dev

# 4. 浏览器访问
# 通常是 http://localhost:3000
```

**群友实战案例（@田超的踩坑过程）：**

```
问题：直接在Cursor打开就运行 npm run dev，报错找不到package.json

原因分析：
- 没有先打开项目文件夹
- 终端在用户主目录，不在项目目录
- 代码生成在 C:\Users\Administrator\realtime-salary-calculator，但终端不在这

解决方案：
1. File → Open Folder
2. 选择生成代码的文件夹（如 realtime-salary-calculator）
3. 再运行 npm run dev
4. 成功！显示 Local: http://localhost:3000
```

**推荐的项目管理方式（@周尹建议）：**
```
1. 提前建好项目文件夹（如：D:\AI_Projects\day01）
2. 用Cursor打开这个空文件夹
3. 再开始和AI对话生成代码
4. 所有文件都会在这个文件夹里，清晰明了
```

## 9. 常见疑难问题Q&A

### Q1: Cursor提示"No response"怎么办？

**完整解决流程：**
```
1. 检查魔法：确保开启且稳定
2. 开启TUN模式：最关键的一步
3. 设置Network为HTTP/1.1
4. 切换节点：避免使用港澳台节点
5. 重启Cursor
6. 如果还不行：询问AI为什么会这样
```

### Q2: npm run dev报错怎么办？

**最常见的原因：**
```
1. 不在项目目录
   解决：用Cursor打开正确的项目文件夹

2. 没安装依赖
   解决：先运行 npm install

3. 端口被占用
   解决：关闭占用端口的程序，或换端口

4. node版本不对
   解决：安装推荐的node版本（通常是LTS版本）
```
